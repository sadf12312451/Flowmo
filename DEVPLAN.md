# 浮墨（Flowmo）开发计划

> **桌面上的生活管家** — 给普通人用的 AI 个人助理

---

## 产品定位

### 一句话定位

**桌面上的生活管家** — 不写代码、不做研究、不当客服，只帮你把生活里那些零碎的事（待办、便签、提醒、习惯）处理好。

### 服务对象

**非技术用户的个人生活管理**（"身边那个最不擅长电脑的人也能用"）。

### 核心差异

| 竞品 | 它们的定位 | 浮墨不做的 |
|---|---|---|
| Cursor / Codex | 程序员的 AI 助手 | 写代码 |
| Claude Desktop | 知识工作者的 AI 助手 | 通用研究/聊天 |
| Open Interpreter | 极客的 AI 助手 | 执行命令、自动化 |
| ChatGPT Desktop | 通用聊天客户端 | 万能对话框 |
| **浮墨** | **普通人的 AI 助手** | **专做生活管理** |

### 不撞车的护城河

1. **本地数据 + AI 的化学反应** — AI 知道你今天的待办、最近的便签
2. **AI 真正动手做事**（Tool Calling）— 不只是回答，能直接创建/修改数据
3. **桌面级深度整合** — 悬浮便签、系统托盘、全局唤起
4. **垂直场景深度** — 番茄钟、习惯打卡、倒计时这些通用 AI 助手做不到
5. **完全本地** — SQLite 数据自主可控，隐私不上云

### 设计原则（写每一行代码前问自己）

> **"你身边那个最不擅长电脑的人，能直接上手用吗？"**

| 决策 | 普通人视角 |
|---|---|
| 默认隐藏 API/Token 配置 | "API"会让他害怕 |
| 模型名改成"快速"/"聪明" | 他不知道 GPT-4o 是什么 |
| 所有操作要"看得见的反馈" | 他需要看到事情真的被记下了 |
| 错误说"网络问题"而非"401" | 他不懂 HTTP 状态码 |
| AI 主动识别意图，不让用户"指挥" | 他不会写提示词 |

## 待解决问题

### P1 — 便签浮窗失焦后无法保持透明/毛玻璃

- **现象**：便签独立浮窗聚焦时有 Acrylic 毛玻璃效果，失焦后效果消失变为纯色背景，且窗口不透明无法看到后面的桌面内容
- **原因**：
  1. Windows Acrylic 效果是系统级行为，失焦时系统会自动禁用模糊
  2. WebView2 动态创建的子窗口即使设置 `transparent: true`，WebView 渲染层仍保持白色底色，导致 CSS `background: transparent` 无效
  3. 通过 JS API (`new WebviewWindow`) 创建的窗口与 `tauri.conf.json` 中预定义的窗口行为不一致
- **可能的解决方向**：
  1. 改用 Rust 端 `WebviewWindowBuilder` 创建浮窗，在 Rust 层直接设置 WebView2 的背景透明
  2. 在 `tauri.conf.json` 中预注册便签窗口模板，由配置驱动窗口创建
  3. 探索 WebView2 的 `put_DefaultBackgroundColor` API 直接设置透明
- **当前状态**：保留聚焦时的 Acrylic 效果，失焦时显示便签颜色纯色背景
- **优先级**：P1（影响用户体验核心功能）
