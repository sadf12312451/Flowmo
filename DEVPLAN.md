# 浮墨（Flowmo）开发计划

> 桌面智能便签应用 — Tauri 2 + Vue 3 + TypeScript + Naive UI

## 技术栈

| 层 | 选型 |
|---|---|
| 桌面框架 | Tauri 2 |
| 前端框架 | Vue 3 + TypeScript |
| 构建工具 | Vite |
| UI 组件库 | Naive UI |
| 状态管理 | Pinia |
| 本地数据库 | SQLite（tauri-plugin-sql） |
| 富文本编辑 | Tiptap |
| 图表 | ECharts |
| AI 接口 | 通用适配层（Claude / OpenAI / Ollama 可切换） |
| 自动更新 | tauri-plugin-updater |
| 多语言 | vue-i18n |

## 待解决问题

### P1 — 便签浮窗失焦后无法保持透明/毛玻璃

- **现象**：便签独立浮窗聚焦时有 Acrylic 毛玻璃效果，失焦后效果消失变为纯色背景，且窗口不透明无法看到后面的桌面内容
- **原因**：
  1. Windows Acrylic 效果是系统级行为，失焦时系统会自动禁用模糊
  2. WebView2 动态创建的子窗口即使设置 `transparent: true`，WebView 渲染层仍保持白色底色，导致 CSS `background: transparent` 无效
  3. 通过 JS API (`new WebviewWindow`) 创建的窗口与 `tauri.conf.json` 中预定义的窗口行为不一致
- **可能的解决方向**：
  1. 改用 Rust 端 `WebviewWindowBuilder` 创建浮窗，在 Rust 层直接设置 WebView2 的背景透明
  2. 在 `tauri.conf.json` 中预注册便签窗口模板，由配置驱动窗口创建
  3. 探索 WebView2 的 `put_DefaultBackgroundColor` API 直接设置透明
- **当前状态**：保留聚焦时的 Acrylic 效果，失焦时显示便签颜色纯色背景
- **优先级**：P1（影响用户体验核心功能）

## 项目结构

```
flowmo/
├── src-tauri/              # Rust 后端
│   ├── src/
│   │   ├── main.rs
│   │   └── lib.rs          # 系统托盘 + 窗口管理 + 毛玻璃命令
│   ├── Cargo.toml
│   └── tauri.conf.json
├── src/                    # Vue 前端
│   ├── components/layout/  # Sidebar, Titlebar
│   ├── views/              # 10个功能页面 + 便签浮窗
│   ├── stores/             # Pinia 状态管理 (notes, todos, countdown, notepad, settings)
│   ├── services/           # db.ts, stickyWindow.ts, ai/
│   ├── i18n/               # 中英文国际化
│   ├── router/             # 路由（含浮窗路由）
│   ├── styles/             # 全局样式
│   └── utils/              # 工具函数
├── package.json
└── vite.config.ts
```
